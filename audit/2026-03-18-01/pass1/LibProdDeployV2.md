## Findings

### A09-1: No Basescan Links for V2 Addresses [INFO]

V1 (`LibProdDeployV1`) annotates every address constant with a Basescan URL comment (e.g., `// https://basescan.org/address/0x...`), which provides an on-demand audit trail directly in the source. V2 omits these links for its four `DEPLOYED_ADDRESS` constants.

While V2 addresses are network-agnostic Zoltu deterministic addresses (making a single-chain Basescan link less definitive), the absence of any on-chain verification link is a minor documentation gap in the audit trail. This is informational only; the values are verified by the test suite against local Zoltu deployment simulation, which is a stronger programmatic guarantee than a URL comment.

No fix file required (INFO severity).
